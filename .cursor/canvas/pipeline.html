⚡ Claude Code ~/.claude/settings.json
Configured via tokenbar-collector configure-claude, which patches ~/.claude/settings.json with OTLP env vars. Claude Code then emits OpenTelemetry metrics over HTTP on every API request. The source is always classified as source_app: "claude_code" with confidence "exact".
CLAUDE_CODE_ENABLE_TELEMETRY=1 OTEL_METRICS_EXPORTER=otlp OTEL_EXPORTER_OTLP_PROTOCOL=http/json OTEL_EXPORTER_OTLP_METRICS_ENDPOINT=http://127.0.0.1:4318/v1/metrics
// also enabled for logs + traces
OTEL_LOGS_EXPORTER=otlp · endpoint /v1/logs
OTEL_TRACES_EXPORTER=otlp · endpoint /v1/traces
HTTP POST · application/json · OTLP metrics payload (nanosecond timestamps)
2
📡 CollectorListener src/collector/listener.ts
Plain Node.js http.createServer bound to 127.0.0.1:4318. Reads the full request body (≤ 5 MB), immediately archives the raw JSON to raw_payloads, then hands the body to the parser. Accepts both a single OTLP object and NDJSON (multiple payloads separated by newlines).
POST /v1/metrics → parse + insert POST /v1/logs → archive only POST /v1/traces → archive only GET /health → 200 + summary JSON
const body = await readBody(req, 5_242_880);
insertRawPayload(db, "metrics", body); // archive first
const payloads = parsePayloadsFromBody(body); // handles NDJSON
const events = payloads.flatMap(p => parseOtlpPayload(p));
insertUsageEvents(db, events);
OtlpPayload[] → parseOtlpPayload() → UsageEvent[]
3
🔬 parseOtlpPayload() src/otlp-parser.ts
Walks resourceMetrics → scopeMetrics → metrics. Groups data points by a composite key of session_id | prompt_id | model | timeUnixNano. Merges four separate token-type data points (input / output / cacheRead / cacheCreation) and the cost data point into a single UsageEvent per request. Also correlates active_time_seconds from a separate claude_code.active_time.total metric series via session+time key.
claude_code.token.usage → input / output / cacheRead / cacheCreation claude_code.cost.usage → cost_usd claude_code.active_time.total → active_time_seconds
// Composite grouping key per unique API request
key = `${sessionId}|${promptId}|${model}|${timeUnixNano}`

// Confidence stamp — only "exact" for official claude-code telemetry
source_confidence: service === "claude-code" ? "exact" : "derived"

// Timestamp: nanoseconds → ISO 8601
nanoToIso(nanoStr) → BigInt(nanoStr) / 1_000_000n → new Date(ms).toISOString()

🍎 tokenbar-macos UsageRepository.swift + SwiftUI MenuBarExtra
Swift 6 menu bar app. Opens the canonical DB path in read-only mode via GRDB's DatabaseQueue. Never writes. Queries are keyed to date(timestamp) for today's totals and window functions for history. The repository returns zero/empty when the DB doesn't exist yet (collector not running).
getTodayTokenTotal() getTodayCostTotal() getTodayInputTokenTotal() getTodayOutputTokenTotal() getSevenDayDailySummaries() getRecentSessions() getModelBreakdown()
// Opened once, read-only, never writes
var config = Configuration()
config.readonly = true
dbQueue = try DatabaseQueue(path: DbPath.canonical, configuration: config)
