# Agent Instructions: TokenBar Phase 0 

## Role Context
You are an expert macOS developer (Swift 6, SwiftUI) and backend engineer (TypeScript/SQLite). You are tasked with building the foundation of "TokenBar", a local-first macOS menu bar app that tracks AI token usage across tools like Claude Code and Cursor. You must work autonomously but stop to ask clarifying questions before making irreversible architectural decisions.

## Project Architecture & Boundary
This project follows an open-core model. 
1. **`tokenbar-core`**: An open-source TypeScript data collector layer that reads local telemetry, maps it to a canonical schema, and writes to a local SQLite database. 
2. **`tokenbar-macos`**: A closed-source Swift 6 macOS application (MenuBarExtra) that reads the local SQLite database to present beautiful, native charts and usage data.

## Phase 0 Objective
Your immediate goal is to prove end-to-end data flow for our most reliable integration (Claude Code) and render it in a native macOS shell. Do not implement Cursor or Claude Desktop yet. Do not implement complex roll-up aggregations or the detailed popover window yet.

## Step-by-Step Execution Plan

### Step 1: Initialize the core schema
- Create the local SQLite database schema based on the `UsageEvent` model defined in `tokenbar-spec.md`.
- Write a simple test or script to insert dummy data to verify the schema.

### Step 2: Build the Claude Code Adapter (TypeScript)
- Locate Claude Code's local OpenTelemetry outputs on the machine.
- Write a TypeScript adapter (`packages/core/adapters/claude-code`) that reads these metrics/events.
- Parse the OTel data (input_tokens, output_tokens, cost) and map it into the canonical SQLite schema.
- Assign `"exact"` as the `source_confidence` label.

### Step 3: Build the SwiftUI Shell (Swift)
- Initialize a macOS App project using Swift 6.
- Implement a `MenuBarExtra` that docks in the macOS menu bar.
- Add an SQLite driver (e.g., GRDB or SQLite.swift) to safely read the database generated by Step 2.
- Display a hardcoded mock value in the menu bar pill (e.g., "CC 12k").

### Step 4: Connect End-to-End
- Wire the SwiftUI menu bar to query the local SQLite database for "Today's total tokens".
- Ensure the menu bar updates dynamically or on a reasonable polling interval when new records are written by the TypeScript collector.

## Constraints & Rules
- **Local-first only:** Do not add any cloud sync, API keys for external dashboards, or Firebase. 
- **Native UI:** Use standard SwiftUI components. Avoid web views or cross-platform wrappers like Electron or Tauri.
- **Privacy:** Do not store actual prompt content or code snippets in the SQLite database, only token counts, costs, and metadata.
- **Confidence Labels:** Every piece of data must have a confidence level attached (`exact`, `derived`, `estimated`).
